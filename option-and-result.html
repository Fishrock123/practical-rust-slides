Absence of Data
Data absence is a continual challenge of interfacing with the real world.
Two forms of data absence:

- Data does not exist
- Processing result was unexpected
"Data does not exist"

- Nullable references - Optional types In some languages this means that all references may be nullable, or that raw pointers may be nullable.
"Processing result was unexpected"

- Error behavior - Exceptional control flow - Result types Mostly C++ exceptional control flow
Rust:

- Option<T>
- Result<T, Err>
These types are not really magic! Rust enums
Rust Enums

Value-Holding
Similar to "Algebraic Data Types" Can hold values for each different enum variation 

pub enum Option>T< {
    /// No value
    None,
    /// Some value of type `T`
    Some(T),
}
    

let a = Some("a");

a; // Option>&str<

a.unwrap(); // &str OR panic
This is very efficient! This compiles to a null reference check! 

let a = Some("a");

match a {
    Some(a) => a,
    None => panic!("Oh No"),
}
    

let a = Some("a");

if let Some(a) {
    // Do something with `a`
}
    

let a = Some("a");

a.is_some(); // true
a.is_none(); // false
    

pub enum Result>T, E< {
    /// Contains the success value
    Ok(T),

    /// Contains the error value
    Err(E),
}
    

let a = Ok("a");

a.is_ok(); // true
a.is_err(); // false

match a {
    Ok(a) => a,
    Err(error) => panic!("Oh No, error: {}", error),
}
